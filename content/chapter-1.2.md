# Chapter 1.2 - Feature List

## 1.2.0 Overview

The following features define the target A.A.S. Field Runtime system built above Hermes:

**Affordance Field:** The environment continuously reads WorldState and exposes a small set of meaningful next moves instead of asking agents to choose raw tools or follow a static graph. \
**WorldState:** A canonical structured representation of the current design world, including goal, intent, project status, branches, artifacts, tensions, commits, moves, risks, open questions, feature state, design debt, trajectory, and recent events. \
**Context Distiller:** A service that converts WorldState, scoped preferences, memory, artifacts, evaluator output, and recent events into compact Agent Briefs so agents receive only the context needed for the selected move. \
**IntentGradient:** A multi-family scoring system that ranks moves by process fit, design gain, search value, execution readiness, user alignment, learning value, governance, elegance, and explicit penalties. \
**MoveCompiler:** A bridge layer that compiles selected A.A.S. moves into Hermes Kanban task groups with profile assignments, dependencies, task packets, output contracts, and artifact write paths. \
**Hermes Bridge:** A service that syncs A.A.S. agent definitions to Hermes profiles, creates Kanban tasks, watches task state/logs/artifacts, translates Hermes events into A.A.S. events, and keeps the Field Navigator live. \
**MoveExecutor:** The typed execution boundary that validates move output, ingests artifacts, updates WorldState, emits events, and coordinates specialized services such as Rhino Compute, GPT Image V2, renderers, and validators. \
**Move Pattern Library:** A data-driven library of reusable architectural tactics built from stable primitives, preconditions, effects, scoring hints, execution templates, validation tests, examples, and success statistics. \
**Move Curator and Sandbox:** Proposed patterns are tested against saved WorldState snapshots, critiqued, deduplicated, promoted, monitored, deprecated, or pruned based on evidence. \
**Feature Registry:** A typed registry of score variables defining each feature's family, range, measurement method, source, confidence, and artifact hooks. \
**Evaluator Contracts:** Plan, section, model, render, board, and concept evaluators measure feature deltas after moves so predicted effects can be compared with actual outcomes. \
**Process Grammar:** A soft phase model for intake, research, concept, branching, ground truth, development, representation, board assembly, QA, and delivery, with phase-fit curves and process landmarks. \
**Trajectory Memory and Design Debt:** The compiler ranks moves against the path already taken, unresolved obligations, repeated failures, missing landmarks, and future unlocks. \
**Elegance Scoring:** Moves are rewarded when they compress complexity, resolve multiple tensions, reuse artifacts, increase coherence, and unlock future work with low fragmentation. \
**Branch Ecology:** Competing design hypotheses can be spawned, developed, critiqued, compared, killed, merged, or committed. \
**Tension Engine:** Design contradictions are tracked as first-class objects linked to artifacts and branches, with resolution moves generated by the affordance compiler. \
**Commitment Ledger:** Major design decisions become project truth only when committed with rationale, evidence, affected artifacts, reversibility, and approval metadata. \
**Preference Store:** User, team, project, session, and agent-profile preferences are scoped, filtered before retrieval, resolved by precedence, and surfaced in Agent Briefs with source manifests. \
**Field Navigator:** The primary UI is a spatial design-field interface showing goals, tensions, branches, moves, commits, artifacts, agents, task state, feature pressures, and lineage as manipulable field objects. \
**Chat Mode:** Chat remains a conversational steering and explanation layer connected to the same WorldState as the Field Navigator. \
**Model Mode:** Ground-truth geometry remains a validation environment. Model state becomes part of WorldState, and Rhino Compute is invoked through typed moves. \
**Trace View:** Trace view shows move history, Hermes task dependency state, logs, artifacts, feature deltas, evaluator results, approvals, and replayable WorldState snapshots. \
**Event Stream:** Stable product events such as `aas.world.updated`, `aas.move.completed`, `aas.tension.created`, `aas.branch.committed`, `aas.commit.created`, and Hermes task translations drive UI updates and audit history.
